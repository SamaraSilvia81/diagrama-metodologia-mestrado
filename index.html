Tema Diretrizes para Implementação de Microfrontends em Larga Escala: Uma Investigação Empírica sobre Desafios Arquiteturais com Ênfase em Dívida Técnica
// Pergunta de pesquisa
"De que forma a Dívida Técnica, enquanto lente de diagnóstico, revela os desafios arquiteturais e de design estruturais em arquiteturas de microfrontend, e quais impactos reverberam na eficiência e sustentabilidade da arquitetura para o desenvolvimento de sistemas de grande escala?"
01
Revisão Sistemática da Literatura
Concluído
RSL · Mapeamento do estado da arte sobre DT, SATD e desafios arquiteturais em MFE. Literatura predominantemente anedótica → identificação da lacuna empírica.
02
Estruturação pelo Framework GQM
Concluído
Goal → Questions → Metrics · Narrativa Promessa → Realidade → Evidência → Impacto → Solução. 5 perguntas investigativas · Métricas via MSR (SATD, complexidade ciclomática, code ownership).
03
Dataset 1 — Caracterização Arquitetural (MSR)
Em andamento
Mapping the Microfrontend Ecosystem: A Dataset of Open-Source Projects
Mineração
API GitHub · busca por heurísticas técnicas · identificação de padrões MFE
Filtragem
Critérios de maturidade · filtros de qualidade finos · validação pós-execução
Deep Mining
Extração de metadados profundos · distribuição de maturidade · curadoria qualitativa
// decisão · restrição de tempo
Recorte Estratégico para Resultados Prévios
Da base de 455 repositórios minerados no Dataset 1, serão selecionados 50–100 projetos como subconjunto representativo para iniciar a construção do Dataset 2 e gerar resultados parciais antes da conclusão do corpus completo — viabilizando análises preliminares dentro do prazo da dissertação.
↓ segue para Dataset 2 com subconjunto
04
Dataset 2 — Indicadores de Saúde (SATD)
Pendente
Construído sobre a base do Dataset 1 · Dívida Técnica Autoadmitida: comentários TODO / FIXME / HACK · complexidade ciclomática · inconsistência de dependências entre módulos · métricas de code ownership
05
Análise Cruzada — Questões de Investigação (QIS)
Pendente
Correlação entre características arquiteturais (monorepo vs multi-repo, estratégias de integração) e indicadores de DT · análise estatística · p-valor · visualização comparativa
06
Proposição das Diretrizes
Pendente
Síntese dos achados empíricos em recomendações práticas estruturadas para desenvolvedores e arquitetos de software · decisões de design informadas em ecossistemas MFE de larga escala
07
Validação das Diretrizes
Pendente
Validação A
Juízo de especialistas · mín. 3 profissionais com expertise reconhecida em arquitetura MFE
Validação B
Aplicação em contexto real · organização parceira · período delimitado · validade externa
08
Contribuição Esperada
Pendente
Diretrizes baseadas em evidências empíricas para arquiteturas MFE em larga escala · abordagem diagnóstica inédita tratando DT como sintoma rastreável · conexão entre saúde arquitetural, experiência do usuário e sustentabilidade dos sistemas
